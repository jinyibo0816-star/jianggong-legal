# /pleading-draft — 要素式民事起诉状起草

## 用途

起草**建设工程施工合同纠纷**的要素式民事起诉状（广东地区法院推行版本），自动填充模板，输出 .docx 到当前项目的 `./output/`。底层调用 MCP 工具 `fill_yaosu_form`。

## 工作原则

**一案一项目**——当前 Cowork 项目根目录就是这个案件的工作目录。读 `./matter.md`，写 `./output/民事起诉状-<日期>.docx`。

## 前置条件

- 当前项目根目录必须存在 `./matter.md`（由 `/matter-intake` 创建）
- 不存在的话，提示用户先跑 `/matter-intake`，然后终止本 skill

## 共享资源路径（绝对路径，不变）

- 要素式空白模板：`/Users/jinyibo/Documents/Claude/Projects/开发/data/templates/民事起诉状.docx`
- 律师档案（通过 MCP `list_lawyer_profiles` 自动读取）
- 知识库（通过 MCP `search_knowledge` 检索）

## 执行步骤

### 第 1 步：读取案件档案

- `pwd` 确认当前在案件项目目录
- 读 `./matter.md` 提取已有信息
- 读 `./_meta.yaml`（如有）拿结构化数据
- 若同目录有 `./chronology.md`、`./时效核查.md` 也一并读取作上下文

### 第 2 步：核对字段完整性

要素式起诉状必填字段（对照 `legal_kb/yaosu_filler/filler.py` 的 `CaseData` dataclass）：

**原告 & 被告主体**
- 名称、住所地、统一社会信用代码、法定代表人、职务、联系电话、所有制性质

**代理人**
- 调用 `list_lawyer_profiles`，让用户挑 1-2 位
- 确认代理权限：一般授权 / 特别授权

**诉讼请求 7 项**（建工领域常见组合，按需勾选）
1. 第 1 项 工程款表述——一段完整文字："判令被告 XX 向原告支付工程款人民币 X 元……"
2. 第 2 项 利息——内容（计算基数/利率/起算日）+ 是否支付至清偿日
3. 第 3 项 优先受偿权——是否主张 + 主张内容（民法典 807 条）
4. 第 4 项 发包人欠付工程款连带责任——是否主张 + 责任主体
5. 第 5 项 赔偿损失
6. 第 13 项 诉讼费用承担（默认勾选）
7. 第 14 项 其他请求
8. 第 15 项 标的总额

**管辖 / 保全 / 鉴定**
- 是否有仲裁/管辖约定 + 内容
- 是否已诉前保全 + 法院/时间/案号
- 是否申请鉴定 + 鉴定事项（造价/质量/工期？）

**事实与理由**（核心，最花时间）
- 合同签订情况（双方主体、合同名称、签订日期）
- 签订主体——发包人 / 承包人 / 出借资质企业 / 实际施工人 四栏
- 建设施工情况（开/竣工、规模、验收）
- 工程款支付——工程总价 / 已支付 / 欠付 / 利息计算
- 工程质量是否合格
- 请求依据——合同约定 + 法律规定（写明条文号）
- 证据清单（先列证据名称，详细证据目录另起一份）

**调解意愿**
- 是否同意诉前调解：是/否/暂不确定（默认"否"）

### 第 3 步：补齐与起草

**逐项检查**——`matter.md` 已有的直接用；缺的逐条问用户。

**事实与理由正文起草**：
1. 用 `search_knowledge` 检索本案争议焦点对应的**法条与裁判规则**（必查关键词如：「建工解释一 第 X 条」、「优先受偿权 行使期限」、「实际施工人 突破合同相对性」等）。
2. 起草一段 800-1500 字的事实与理由叙述，结构：
   - 第一段：合同关系与履约事实（双方、合同、开竣工、验收）
   - 第二段：争议的具体情况（欠款数额、质量/工期/造价争议事实）
   - 第三段：催告与协商经过（尤其重要——影响时效抗辩与利息起算）
   - 第四段：法律依据小结（民法典合同编 + 建工解释一相关条文，**必须把条文号写准确**）
3. **草稿给用户审核**，让用户修改后再进入下一步。**禁止直接生成 docx 后再说"请审核"**。

**法律依据引用规范**：
- 民法典：写"《中华人民共和国民法典》第 X 条"
- 建工解释一：写"《最高人民法院关于审理建设工程施工合同纠纷案件适用法律问题的解释（一）》第 X 条"
- 引用前用 `search_knowledge` 在知识库里核对一次条文实际内容——条文号背错会直接被法官看出来。

## 重要约束

- **诉讼请求金额必须用户明确确认，不要从合同价款自动推算**。欠付金额、利息基数、利息利率、起算日都是律师判断，不是机器推算。
- **优先受偿权（民法典 807 条）只在符合主张条件时勾选**：必须是承包人主张、必须在工程竣工日/合同约定竣工日起 18 个月内、必须未受清偿。条件不齐时主动告诉用户为什么不能勾。
- **实际施工人路径下**（建工解释一第 43 条）：突破合同相对性向发包人主张，必须把"实际施工人""转包/违法分包链""发包人欠付转包人款项"三个事实写清楚，不能含糊。
- **法条号必须核对**——民法典合同编建工部分集中在 788-808 条，建工解释一是 2020 年修正版（不是 2004 版）。引用前 `search_knowledge` 验证。
- **不要写到 `/开发/output/`**——所有输出必须落在当前项目的 `./output/` 下。
- 任何输出都是律师审核前的草稿，**禁止跳过律师人工确认直接发送或盖章**。
