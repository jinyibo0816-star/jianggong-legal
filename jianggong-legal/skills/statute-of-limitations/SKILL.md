# /statute-of-limitations — 建工诉讼时效核查

## 用途

针对建设工程纠纷的**多套并行期间制度**逐项核查，给出每项请求权的「在效 / 临界 / 过期」判断和依据。建工纠纷败诉最常见原因之一就是时效，不能漏。

## 工作原则

**一案一项目**——当前 Cowork 项目根目录就是这个案件的工作目录。

- 输入：读 `./matter.md`（必需）+ `./chronology.md`/`./chronology.yaml`（如有）
- 输出：写 `./时效核查.md`
- 操作日志追加到 `./_history.md`

## 核查维度（建工领域）

针对当前案件涉及的请求权，**只核查相关的几项**，不要全部都核——浪费时间。

### A. 普通诉讼时效（民法典 188 条）—— 3 年
- **起算点**：权利人知道或应当知道权利受损害且义务人之日。
- 建工应用：工程款主张的起算点通常以下三选一（看证据）：
  - 约定的付款日（合同/补充协议明确）
  - 结算完成且发包人逾期未付
  - 验收合格且工程款未付（无明确付款节点时）
- **中断事由**：催告（书面/可证明的电子证据）、起诉、达成还款协议、对方部分履行
- 输出：起算日 → 累计经过时间 → 中断次数 → 最近一次中断日 → 剩余时间

### B. 最长权利保护期（民法典 188 条第 2 款）—— 20 年
- 自权利受损害之日起算，**不可中断**。建工案件极少触发，但跨度大的旧案要查。

### C. 建工质量保修责任（建工解释一第 19 条 + 建设工程质量管理条例第 40 条）
- 不同部位有最低保修期：
  - 地基基础、主体结构：设计文件规定的合理使用年限
  - 屋面防水、有防水要求的卫生间/房间/外墙面：5 年
  - 供热与供冷系统：2 个采暖期、供冷期
  - 电气管线、给排水管道、设备安装、装修：2 年
- **起算点**：竣工验收合格之日
- 业主在保修期内发现质量问题 → 可以主张修复
- 业主在保修期**届满后**发现的隐蔽质量问题——可在合理期限内主张，但举证难度大

### D. 工程款优先受偿权（民法典 807 条 + 建工解释一第 41 条）—— 18 个月
- **行使期限**：自发包人应当给付建设工程价款之日起 **18 个月**
- 这是**除斥期间，不可中断**。过了就是过了。
- "应当给付价款之日"通常是：合同约定的最后付款日 / 工程款全部结清的约定日；无约定时按建工解释一第 27 条规定的几个推定日。
- 行使方式：以书面通知、诉讼、申请仲裁等明确方式主张
- **核查重点**：① 我方是不是承包人（实际施工人能否主张？建工解释一第 41 条没明确赋予，目前主流不支持） ② 行使期限是否过期 ③ 是否在工程未灭失/未转让范围内

### E. 农民工工资专项时效
- 民法典 196 条排除：劳动报酬请求权**不适用 3 年时效**——但建工里实际施工人/包工头主张的"工程款中的农民工工资部分"不是劳动法上的劳动报酬，仍适用 3 年。
- 务工人员个人通过农民工工资保证金/专项程序主张，按相关法规。

### F. 质量异议期（合同约定 + 建工解释一第 14 条）
- 合同约定的发现质量缺陷后通知期限——逾期视为认可
- 没有约定时按合理期限

### G. 仲裁时效
- 同 3 年。如合同有仲裁条款，要在仲裁规则项下另行计算。

### H. 诉前/诉中调解协议履行
- 司法确认/调解书生效后申请执行的期限：2 年（民事诉讼法 246 条）

## 执行步骤

### 第 1 步：读取案件档案与时间线

- `pwd` 确认当前在案件项目目录
- 读 `./matter.md`（不存在则提示用户先跑 `/matter-intake`）
- 若存在 `./chronology.md` / `./chronology.yaml`，一并读取
- 提取关键日期：
  - 合同约定付款日
  - 竣工/验收日
  - 结算完成日
  - 最后一次催告日（及证据形式）
  - 拟起诉日（或今天）

### 第 2 步：确定要核查的维度

根据案件争议焦点和我方角色，**只核查相关项**：

| 我方主张                         | 必查维度          |
|----------------------------------|-------------------|
| 工程款                           | A（必查）         |
| 工程款 + 优先受偿权              | A + D（必查）     |
| 工程质量问题修复 / 损失赔偿      | A + C（必查） + F |
| 农民工工资                      | E + A             |
| 利息                            | A（随主债）       |
| 违约金                          | A                 |

### 第 3 步：逐项核查并起草报告

对每个适用维度，调用 `search_knowledge` 检索一次本所知识库里关于该问题的**最新裁判规则**和注释（关键词如「优先受偿权 行使期限 起算」、「实际施工人 优先受偿权」、「时效中断 催告 证据」），引用 1-2 条到报告里作为支撑。

**报告结构**（写入 `./时效核查.md`）：

```markdown
# 时效核查报告 — <案件简称>

核查日期：<YYYY-MM-DD>
核查人：<律师姓名>

## 核查结论摘要

| 请求权                | 期间制度          | 起算日       | 关键节点          | 结论        |
|-----------------------|-------------------|--------------|-------------------|-------------|
| 工程款本金            | 3 年普通时效      | YYYY-MM-DD   | 中断于 YYYY-MM-DD | ✅ 在效     |
| 利息                  | 随主债            | -            | -                 | ✅ 在效     |
| 优先受偿权            | 18 个月除斥       | YYYY-MM-DD   | 18 个月届满 YYYY  | ⚠️ 临界 / ❌ 过期 |
| ...                  | ...               | ...          | ...               | ...         |

## 逐项分析

### 1. 工程款本金（普通诉讼时效 3 年）

**起算依据**：[合同 X 条约定付款日 / 结算后逾期 / ...]

**累计经过**：X 年 X 月
**中断节点**：
  - YYYY-MM-DD 书面催告（《催款函》快递回执编号 XX）—— **有效中断**
  - YYYY-MM-DD 对方回函承认欠款 —— **构成承认**
**最近重新起算日**：YYYY-MM-DD
**至拟起诉日剩余**：X 月 X 日
**结论**：✅ 在效
**风险点**：[催告证据是否充分？是否能证明对方收到？]
**依据条文**：民法典第 188 条、第 192 条、第 195 条

### 2. 优先受偿权（除斥 18 个月）

**起算依据**：…
（结构同上）

…
```

### 第 4 步：风险提示与建议

报告末尾追加：

```markdown
## 综合建议

1. **必须在 YYYY-MM-DD 前完成的动作**：[列出最近的临界点]
2. **证据补强建议**：[催告证据是否需要补充？]
3. **起诉时点建议**：[基于多维度综合判断的建议起诉窗口]

> ⚠️ 本报告基于现有案件档案与律师录入信息，不构成最终法律判断。建议由承办律师核对所有时间节点的证据真实性后采纳。
```

### 第 5 步：归档

写完报告后，在 `./_history.md` 追加一行：

```
- <YYYY-MM-DD HH:MM> 完成时效核查 → 时效核查.md
```

## 重要约束

- **所有产出落在 `./`**（当前项目根），不要写到任何其他绝对路径。
- **不要瞎填日期**：用户没明确给出的日期标记为「待核实」，不要从其他字段反推。
- **优先受偿权 18 个月是除斥期间，不是诉讼时效**——过了就不能行使，没有中断。这一点务必在报告里清楚区分。
- **催告证据形式要细问**：口头催告基本无效；微信/短信需保留原始记录及实名认证；EMS 需快递回执；公证送达最稳。如果只有口头催告，明确告诉用户中断可能不成立。
- **不替律师下"应当起诉"或"已经过时效不能起诉"的最终结论**——只输出核查事实和风险提示，最终判断让承办律师做。
- 涉及农民工工资的，建议同时提示用户考虑民法典 196 条第 1 项的影响。
